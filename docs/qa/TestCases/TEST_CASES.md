# Тест-кейсы: OnlineCourses

## TC-01 Регистрация нового пользователя

Описание:

Проверка успешной регистрации нового пользователя в системе.

Предусловия:

Приложение API запущено. Пользователь не авторизован. Email нового пользователя еще не существует в базе данных.

Последовательность действий:

1. Открыть Swagger по адресу `http://localhost:5064/swagger` или Postman.
2. Выбрать запрос `POST /api/Auth/register`.
3. Ввести тело запроса:

```json
{
  "email": "testuser@example.com",
  "password": "password123",
  "fullName": "Тест Пользователь"
}
```

4. Отправить запрос.

Ожидаемый результат:

Сообщение `User registered successfully`.

Код ответа:

```text
200 OK
```

## TC-06 Создание курса преподавателем

Описание:

Проверка успешного создания курса пользователем с ролью `teacher`.

Предусловия:

Приложение API запущено. Пользователь `teacher@local.dev` авторизован и имеет `accessToken`.

Последовательность действий:

1. Открыть Swagger или Postman.
2. Выбрать запрос `POST /api/Courses`.
3. Добавить заголовок:

```text
Authorization: Bearer {teacherAccessToken}
```

4. Ввести тело запроса:

```json
{
  "title": "Основы ручного тестирования",
  "description": "Курс для практики ручного тестирования",
  "price": 990,
  "level": "beginner",
  "categoryId": 1,
  "coverImageUrl": null
}
```

5. Отправить запрос.

Ожидаемый результат:

Курс успешно создается. В ответе возвращается объект созданного курса.

Код ответа:

```text
201 Created
```

## TC-12 Проверка загрузки недопустимого файла аватара

Описание:

Проверка валидации расширения файла при загрузке аватара пользователя.

Предусловия:

Приложение API запущено. Пользователь авторизован. Подготовлен файл с недопустимым расширением, например `test.exe`.

Последовательность действий:

1. Открыть Swagger или Postman.
2. Выбрать запрос `POST /api/files/avatar`.
3. Добавить заголовок:

```text
Authorization: Bearer {accessToken}
```

4. В `multipart/form-data` выбрать файл `test.exe`.
5. Отправить запрос.

Ожидаемый результат:

Система отклоняет загрузку файла, так как расширение не входит в список разрешенных.

Код ответа:

```text
400 Bad Request
```

```text

Screenshots/TC-12-step2.png
```
