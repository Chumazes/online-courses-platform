# Примеры баг-репортов

## BUG-001: Документация API отображалась с битой кодировкой

Серьезность: Medium

Приоритет: High

Окружение:

- Windows 10
- Репозиторий: `C:\OSTALNOE\kursovka`

Шаги воспроизведения:

1. Открыть `API_DOCUMENTATION.md`.
2. Проверить русский текст.

Фактический результат:

- Русский текст отображается символами вида `Рџ`, `Рґ`, `Рµ`.

Ожидаемый результат:

- Русский текст отображается корректно в UTF-8.

Статус:

- Исправлено: документация переписана в UTF-8.

## BUG-002: README был пустым

Серьезность: Medium

Приоритет: High

Шаги воспроизведения:

1. Открыть `README.md`.

Фактический результат:

- README не содержит полезной информации о проекте.

Ожидаемый результат:

- README содержит описание проекта, стек, команды запуска, команды тестов, информацию о БД и тестовых пользователях.

Статус:

- Исправлено.

## BUG-003: CI может падать на Ubuntu из-за Windows-target тестового проекта

Серьезность: High

Приоритет: Medium

Окружение:

- GitHub Actions
- Runner: `ubuntu-latest`
- Тестовый проект: `net8.0-windows`

Шаги воспроизведения:

1. Отправить изменения в GitHub.
2. Дождаться запуска workflow `.NET CI`.

Фактический результат:

- Workflow может упасть, потому что тестовый проект использует Windows-specific target framework.

Ожидаемый результат:

- CI должен запускаться на `windows-latest`, либо тесты должны быть разделены на платформонезависимые API-тесты и Windows UI-тесты.

Рекомендация по исправлению:

- Сменить runner на `windows-latest`, либо создать отдельный проект `OnlineCourses.Api.Tests` с target framework `net8.0`.

## BUG-004: В некоторых ответах API есть сообщения с битой кодировкой

Серьезность: Low

Приоритет: Medium

Шаги воспроизведения:

1. Вызвать ошибки валидации категорий в `CoursesController`.
2. Проверить текст сообщения в ответе.

Фактический результат:

- Некоторые русские сообщения могут возвращаться с битой кодировкой.

Ожидаемый результат:

- API возвращает читаемые UTF-8 сообщения или единый английский текст.

Рекомендация по исправлению:

- Переписать битые response-сообщения в контроллерах в UTF-8.
- Сделать это отдельным коммитом по очистке API.

## BUG-005: SQLite-файл не должен случайно попадать в коммит

Серьезность: Low

Приоритет: Medium

Шаги воспроизведения:

1. Запустить API локально.
2. Проверить `git status`.

Фактический результат:

- `OnlineCourses.API/online-courses-dev.db` отображается как untracked-файл.

Ожидаемый результат:

- База должна генерироваться локально и игнорироваться git, либо коммититься только если преподаватель явно просит готовый файл БД.

Рекомендация по исправлению:

- Добавить `*.db` или `OnlineCourses.API/online-courses-dev.db` в `.gitignore`, либо хранить SQL seed-скрипт вместо файла БД.
