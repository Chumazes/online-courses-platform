# QA-чек-лист: OnlineCourses

## Smoke-проверка

- [ ] API запускается командой `dotnet run --launch-profile http`.
- [ ] Swagger доступен по адресу `http://localhost:5064/swagger`.
- [ ] SQLite-база `OnlineCourses.API/online-courses-dev.db` создается автоматически.
- [ ] `GET /api/Courses` возвращает список курсов без авторизации.
- [ ] `GET /api/Courses/categories` возвращает список категорий без авторизации.
- [ ] Пользователь `student@local.dev / 123456` успешно входит в систему.
- [ ] Пользователь `teacher@local.dev / 123456` успешно входит в систему.
- [ ] Пользователь `admin@local.dev / 123456` успешно входит в систему.
- [ ] `GET /api/Auth/me` с токеном возвращает профиль пользователя.
- [ ] `GET /api/Auth/me` без токена возвращает `401 Unauthorized`.
- [ ] Desktop/Web-клиент открывает каталог курсов.

## Авторизация и регистрация

- [ ] Регистрация нового пользователя с уникальным email проходит успешно.
- [ ] Повторная регистрация с тем же email возвращает `400 Bad Request`.
- [ ] Вход с правильными данными возвращает access token и refresh token.
- [ ] Вход с неверным паролем возвращает `401 Unauthorized`.
- [ ] Refresh token обновляет пару токенов.
- [ ] Logout отзывает текущий refresh token.
- [ ] Logout all отзывает все refresh token пользователя.

## Роли и доступы

- [ ] `student` не может создавать курс.
- [ ] `teacher` может создать курс.
- [ ] `teacher` может редактировать только свой курс.
- [ ] `teacher` не может редактировать чужой курс.
- [ ] `admin` может управлять категориями.
- [ ] `admin` может модерировать отзывы.
- [ ] Защищенные POST/PUT/DELETE endpoint без токена возвращают `401`.
- [ ] Запрос с недостаточной ролью возвращает `403`.

## Курсы

- [ ] Список курсов поддерживает пагинацию.
- [ ] Фильтр по уровню курса работает.
- [ ] Фильтр по категории работает.
- [ ] Поиск по названию или описанию работает.
- [ ] Сортировка по рейтингу работает.
- [ ] `GET /api/Courses/{id}` возвращает существующий курс.
- [ ] `GET /api/Courses/{id}` для несуществующего курса возвращает `404`.
- [ ] Создание курса с валидным телом возвращает `201 Created`.
- [ ] Обновление курса меняет поля курса.
- [ ] Удаление курса удаляет курс из списка.

## Разделы и уроки

- [ ] `GET /api/courses/{courseId}/sections` возвращает разделы курса.
- [ ] Создание раздела доступно преподавателю или администратору.
- [ ] Обновление раздела сохраняет новое название и описание.
- [ ] Удаление раздела удаляет связанные уроки или корректно обрабатывает ограничение БД.
- [ ] `GET /api/sections/{sectionId}/lessons` возвращает уроки раздела.
- [ ] Создание урока доступно преподавателю или администратору.
- [ ] Обновление урока меняет название, тип, порядок и контент.
- [ ] Удаление урока удаляет урок из раздела.

## Запись на курсы

- [ ] Авторизованный студент может записаться на опубликованный курс.
- [ ] Повторная запись на тот же курс возвращает ошибку.
- [ ] `GET /api/Enrollments/my` возвращает курсы текущего пользователя.
- [ ] Пользователь может отписаться от курса.
- [ ] Преподаватель может посмотреть студентов своего курса.
- [ ] Студент не может посмотреть список студентов курса.

## Прогресс

- [ ] Прогресс по уроку создается при первом обновлении.
- [ ] Повторное обновление прогресса меняет `isCompleted` и `watchTime`.
- [ ] Прогресс курса пересчитывается по завершенным урокам.
- [ ] Нельзя обновить прогресс по курсу, на который пользователь не записан.
- [ ] `GET /api/Progress/my` возвращает прогресс по всем курсам пользователя.

## Отзывы

- [ ] Пользователь может оставить отзыв к курсу.
- [ ] Повторный отзыв к тому же курсу от того же пользователя запрещен.
- [ ] Новый отзыв требует модерации.
- [ ] Публичный список отзывов показывает только одобренные отзывы.
- [ ] Администратор может открыть список отзывов для модерации.
- [ ] Администратор может одобрить отзыв.
- [ ] Рейтинг курса пересчитывается после изменения отзывов.

## Файлы

- [ ] Авторизованный пользователь может загрузить аватар допустимого расширения.
- [ ] Загрузка аватара недопустимого расширения отклоняется.
- [ ] Загрузка слишком большого аватара отклоняется.
- [ ] Преподаватель может загрузить файл к своему уроку.
- [ ] Студент не может загрузить файл к уроку.
- [ ] Файл урока можно скачать через `GET /api/files/download`.
- [ ] Скачивание несуществующего файла возвращает ошибку.

## Регрессия

- [ ] После создания курса он отображается в общем списке.
- [ ] После изменения курса сбрасывается кэш списка курсов.
- [ ] После удаления курса он не открывается по ID.
- [ ] После создания категории она доступна при создании курса.
- [ ] После удаления категории курсы не теряют целостность данных.
- [ ] После обновления профиля новые данные видны в `GET /api/Auth/me`.
- [ ] После перезапуска API SQLite-данные сохраняются.
